Planejamento do Projeto

⦁	Definir os sinais vitais a serem monitorados (ex.: batimento cardíaco, temperatura corporal, oxigenação).

⦁	Escolher sensores compatíveis com ESP32 (ex.: Pulse Sensor, DHT22, MAX30102).

⦁	Definir os limites para alertas (ex.: batimento > 180 bpm, temperatura > 39°C).

Simulação no Wokwi

⦁	Criar o circuito no Wokwi com ESP32 + sensores simulados.

⦁	Programar o código em Arduino IDE para leitura dos sensores.

⦁	Implementar lógica de alerta local (exibir no Serial Monitor quando valores estão fora do normal).

Edge Computing no ESP32

⦁	Processar os dados localmente (não apenas enviar brutos).

⦁	Criar função para detectar situação normal/anormal.

⦁	Gerar mensagem de alerta (“ALERTA: Jogadora com risco”).

Integração IoT

⦁	Configurar o ESP32 para enviar dados via MQTT/HTTP para a VM na Azure.

⦁	Estruturar payload em JSON (ex.: { "batimento": 175, "status": "alerta" }).

⦁	Configuração da VM + FIWARE

⦁	Criar VM na Azure.

⦁	Instalar Docker + FIWARE Orion Context Broker.

⦁	Configurar entidades (Jogadora, Sinais Vitais).

⦁	Garantir comunicação entre ESP32 e FIWARE (teste com Postman).
